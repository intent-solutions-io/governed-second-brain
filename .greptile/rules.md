# governed-second-brain (umbrella / landing) — review context for Greptile

This is the **umbrella / landing repo** for the Governed Second Brain ("Bob's Big Brain") stack. It
holds **ecosystem-level documentation + cross-repo working-surface tooling** — the landing README (the
primary deliverable / product page), governance files, `assets/`, `repos.yml`, the `bin/` helpers
(`gsb`, `teamkb-systemmap.sh`), and the aggregated `changelogs/`. **No application code lives here** —
code, tests, and PRs belong in the engine repos (ICO, INTKB) and the plugin repo.

So reviews here are about **claim honesty and doc drift**, not code correctness.

## What you must protect

1. **The honesty box (`README.md`).** The README carries a "What the receipt does *not* do" trust box.
   The chain is **tamper-evident**, not tamper-proof. **Forbidden as product claims:** tamper-proof,
   immutable, non-repudiation (local mode), blockchain. Local mode = integrity + ordering +
   rewrite-detection; cross-actor attribution needs the pushed anchor + per-actor signatures.
2. **The thesis framing.** The competitive axis is **govern + receipts, not recall**. Preserve
   compile -> spool -> govern -> receipts and "the model proposes; the deterministic system owns
   durable state and control." Don't soften "deterministic / append-only / hash-chained" into vague
   memory-marketing language.
3. **Don't conflate the two distribution channels** (public plugin vs private team marketplace), and
   **R2 is off-host backup only** — NOT the team bridge (the bridge is the tailnet INTKB API + the
   plugin's team mode).
4. **The spool trust boundary in prose.** Docs must not claim the model governs / promotes / writes
   audit events, or that `candidates` (insert-only source of truth, 005-AT-ARCH) is mutable.
5. **Never re-hash the chain (D5).** The 155 CHAIN_FORK breaks are benign; ok:false = real tamper.
6. **Generated content is off-limits to hand edits:** the `<!-- AUTOGEN:live-stats -->` block in
   005-AT-ARCH (regenerated by `bin/teamkb-systemmap.sh`) and the whole `changelogs/` mirror
   (regenerated by `scripts/aggregate-changelogs.sh` from each repo's own CHANGELOG.md).
7. **Doc filing:** new docs are `NNN-CC-ABCD-description.md` under `000-docs/` with one global sequence.
8. **No plaintext secrets** in `bin/`, scripts, or manifests; `~/.teamkb/tokens.json` is a SECRET.

## Prioritize

- Claim honesty (forbidden words, over-claim, softened trust box).
- Thesis / architecture drift between the README and 005-AT-ARCH / 007-AT-SMAP.
- Hand-edits to generated content (live-stats block, changelogs/).

## Deprioritize

- Prose style / wording taste on non-claim text.
- Churn on `changelogs/` and `assets/*.png` (both regenerated).

team marketplace + this repo) plus the live `~/.teamkb` data dir — see `repos.yml`, `000-docs/007-AT-SMAP`,
